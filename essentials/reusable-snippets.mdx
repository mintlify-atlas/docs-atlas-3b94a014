Reusable snippets help you maintain consistency across your documentation by allowing you to write content once and reference it in multiple places. Any updates to the snippet automatically propagate to all pages that use it.

<Note>
All snippet files must be placed in the `snippets` directory. Any page in this directory will be treated as a snippet and will not render as a standalone page.
</Note>

## Creating a snippet

### Basic snippet

Create a new `.mdx` file in the `snippets` directory with your reusable content.

**Pre-condition**: You must create your snippet file in the `snippets` directory.

```mdx snippets/my-snippet.mdx
Hello world! This is my content I want to reuse across pages.
```

Then import and use it in any page:

```mdx destination-file.mdx
---
title: My Page
description: My Description
---

import MySnippet from '/snippets/my-snippet.mdx';

## Header

Lorem ipsum dolor sit amet.

<MySnippet />
```

### Snippets with variables

You can create dynamic snippets that accept props:

```mdx snippets/greeting.mdx
Hello world! This is my content I want to reuse across pages. My keyword of the day is {word}.
```

Use the snippet with custom values:

```mdx destination-file.mdx
---
title: My Page
description: My Description
---

import Greeting from '/snippets/greeting.mdx';

## Welcome

<Greeting word="documentation" />
```

<Warning>
The content that you want to reuse must be inside the `snippets` directory in order for the import to work.
</Warning>

## Reusable variables

Export constant values from snippets to maintain consistency across your documentation.

### Step 1: Export variables

Create a snippet file with exported variables:

```mdx snippets/custom-variables.mdx
export const companyName = 'Acme Corp';

export const supportEmail = 'support@acme.com';

export const config = { 
  apiVersion: 'v2',
  baseUrl: 'https://api.acme.com'
};
```

### Step 2: Import and use variables

Import the variables in your destination file:

```mdx destination-file.mdx
---
title: Contact Us
description: Get in touch with our team
---

import { companyName, supportEmail, config } from '/snippets/custom-variables.mdx';

## Support

Welcome to {companyName}! For support, email us at {supportEmail}.

Our API is available at {config.baseUrl}/{config.apiVersion}.
```

## Reusable components

Create complex reusable components using arrow functions.

### Step 1: Create a component

Inside your snippet file, export a component that accepts props:

```mdx snippets/custom-component.mdx
export const FeatureCard = ({ title, description, icon }) => (
  <div style={{ padding: '20px', border: '1px solid #e5e7eb', borderRadius: '8px' }}>
    <h3>{icon} {title}</h3>
    <p>{description}</p>
  </div>
);
```

<Warning>
MDX does not compile inside the body of an arrow function. Stick to HTML syntax when you can or use a default export if you need to use MDX components.
</Warning>

### Step 2: Import and use the component

Import the component and pass in props:

```mdx destination-file.mdx
---
title: Features
description: Product features
---

import { FeatureCard } from '/snippets/custom-component.mdx';

## Our Features

<FeatureCard 
  icon="🚀"
  title="Fast Performance" 
  description="Lightning-fast response times for all operations."
/>

<FeatureCard 
  icon="🔒"
  title="Secure by Default" 
  description="Enterprise-grade security built into every layer."
/>
```
